# Chrome DevTools MCP 代码同步工具使用指南

## 功能说明

这个工具允许你通过 Chrome DevTools MCP 修改页面元素，然后将修改同步回项目源代码。

## 使用方法

### 1. 修改文本内容

**示例：修改标题 "New Collection"**

1. 告诉我你想修改的元素（例如："我想把标题改成'最新系列'"）
2. 我会通过 DevTools 获取当前元素
3. 我会修改对应的源代码文件（`app/page.tsx`）

**支持的文本修改：**
- 页面标题（h1, h2, h3 等）
- 按钮文字
- 链接文字
- 描述文字

### 2. 修改样式

**示例：修改颜色、字体大小、间距等**

1. 告诉我你想修改的样式（例如："我想把标题颜色改成红色"）
2. 我会定位到对应的样式代码
3. 我会修改内联样式或 CSS 文件

**支持的样式修改：**
- 颜色（color, backgroundColor）
- 字体大小（fontSize）
- 间距（margin, padding）
- 布局（display, gridTemplateColumns）

### 3. 修改图片

**示例：更换商品图片**

1. 告诉我你想修改的图片（例如："我想把经典白衬衫的图片改成 /images/new-shirt.jpg"）
2. 我会修改对应的图片路径
3. 如果图片在数据库中，我会提示你修改数据库或 seed 文件

**支持的图片修改：**
- 商品图片路径
- 图片样式（大小、圆角等）

### 4. 修改功能

**示例：添加新功能、修改交互逻辑**

1. 告诉我你想修改的功能（例如："我想在商品卡片上添加一个收藏按钮"）
2. 我会分析代码结构
3. 我会修改对应的组件文件

## 注意事项

1. **数据驱动的元素**：商品名称、描述、图片等来自数据库，修改这些需要：
   - 修改 `prisma/seed.ts`（重新 seed）
   - 或通过 Prisma Studio 直接修改数据库
   - 或修改 API 返回的数据格式

2. **样式修改**：
   - 内联样式：直接修改组件文件中的 `style` 属性
   - 全局样式：修改 `app/globals.css`

3. **功能修改**：
   - 需要修改 React 组件逻辑
   - 可能需要修改 API 路由
   - 可能需要修改数据库结构

## 使用示例

### 示例 1：修改标题

**你说：** "我想把 'New Collection' 改成 '最新系列'"

**我会：**
1. 通过 DevTools 确认元素位置
2. 修改 `app/page.tsx` 第 43 行
3. 将 `New Collection` 改为 `最新系列`

### 示例 2：修改标题颜色

**你说：** "我想把标题颜色改成蓝色 #0066cc"

**我会：**
1. 通过 DevTools 确认元素
2. 修改 `app/page.tsx` 第 43 行的 style 属性
3. 添加或修改 `color: '#0066cc'`

### 示例 3：修改商品图片路径

**你说：** "我想把经典白衬衫的图片改成 /images/new-shirt.jpg"

**我会：**
1. 检查商品数据来源（数据库）
2. 修改 `prisma/seed.ts` 中对应的 `imageUrl`
3. 或者提示你通过 Prisma Studio 修改数据库

## 开始使用

现在你可以告诉我你想修改什么，我会：
1. 通过 Chrome DevTools 获取当前页面状态
2. 定位到对应的源代码位置
3. 进行修改
4. 确认修改结果
